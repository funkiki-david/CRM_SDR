## 一、这个系统做什么

一个专为 SDR 销售员设计的轻量 CRM，核心能力：

- 从 Apollo.io 搜人、导入、自动补全资料
- 每天快速记录联系活动（支持语音录入）
- AI 自动生成客户研究报告 + 个性化邮件
- 向量数据库驱动的语义搜索（用大白话搜历史记录）
- 多 Gmail 邮箱集成，系统内直接发冷邮件
- 三级权限管理（Admin / Manager / SDR）

## 二、页面 & 功能清单

### 2.1 首页 `/dashboard`

**设计方向**：纯白/浅色底，不要花哨图表和看板。打开就是一个行动清单。

首页内容：
- **今日 Follow-up 清单** — SDR 登录后第一眼看到的就是今天该跟进谁
  - 每条显示：客户名字、上次联系时间、一句话摘要、建议下一步动作
  - 按紧急程度排序
  - 点一下进入联系人详情或直接快速录入
- **团队 Activity Feed** — 按时间顺序显示团队动态
  - 类似社交媒体 feed：「David 给 John Smith 发了冷邮件」「Lisa 跟 Sarah Lee 通了电话」
  - 不是看板拖拽，是简单的时间流

### 2.2 联系人列表 `/contacts`

- 左右分栏布局：左 30% 联系人列表（可搜索滚动），右 70% 显示选中的人的详情
- 搜索 + 筛选功能

### 2.3 联系人详情 `/contacts/[id]`

**客制化重点 — 深度客户画像：**

| 区块 | 内容 |
|------|------|
| 基本信息 | 姓名、职位、公司、邮箱、电话 |
| LinkedIn | 直接链接到他的 LinkedIn 主页 |
| AI 人物研究报告 | 基于 LinkedIn 履历 + 公开信息自动生成的个人画像 |
| AI 公司研究报告 | 针对该客户公司网站的分析（产品、定位、痛点） |
| 行业关键词标签 | AI 自动提取（后续 David 会给详细模板） |
| 活动时间线 | 所有历史联系记录，按时间倒序，每条有图标标识类型 |
| AI 建议面板 | 基于历史互动给出的跟进建议 |

### 2.4 快速录入 `/activities/new`

**客制化重点 — 语音录入：**

- 基本流程：选人 → 选类型（电话/邮件/LinkedIn/会议）→ 写备注 → 完成
- **语音录入**：点麦克风按钮，对着说话，系统自动转文字
- **AI 语音解析（进阶）**：从语音内容自动识别联系人、活动类型、关键信息、下次跟进日期
  - 例：说"今天跟 John 打了电话，他说 Q3 预算没批，让我下个月再联系"
  - 系统自动填好：联系人=John、类型=电话、下次跟进=下个月

### 2.5 Apollo 搜人 `/finder`

- ICP 筛选表单（职位、行业、公司规模、地区）
- 搜索结果列表
- **去重检测**：搜出来的人如果本地已存在，显示蓝色「已存在」标签 + 上次联系时间；新人显示绿色「新线索」
- 一键导入：新联系人创建记录，已存在的更新信息（不覆盖手动填的备注）
- 导入结果报告：「新增 12 人，更新 3 人」

### 2.6 AI 邮件起草 `/ai-compose`

- 选联系人 → 系统自动调取他的研究报告 + 历史互动 → AI 生成个性化邮件
- 选择从哪个邮箱发出
- 可手动编辑后发送

### 2.7 语义搜索 `/ai-search`

- 自然语言搜索所有历史记录
- 例："哪些客户提到过预算问题" → 返回所有相关对话

### 2.8 设置 `/settings`

- Apollo API Key 配置
- Gmail 邮箱管理（绑定/解绑）
- 团队成员管理
- 邮件模板库

## 三、邮件系统

**客制化重点 — 完整冷邮件闭环：**

| 功能 | 说明 |
|------|------|
| 多邮箱绑定 | 一个 SDR 可接入多个 Gmail（如公司邮箱 + cold outreach 专用邮箱） |
| 系统内发冷邮件 | 不用跳出去开 Gmail，在联系人详情页直接选邮箱发送 |
| 邮件模板库 | 存几套常用的 cold email 模板，一键套用 |
| AI 个性化改写 | 选模板后 AI 根据客户研究报告自动改写开头 |
| 发送追踪 | 对方有没有打开邮件、有没有点链接 |
| 收件同步 | 客户回信了自动出现在活动时间线里 |

**完整流程**：找人 → 研究 → 写信 → 发出 → 追踪 → 回信自动记录 → AI 建议下一步

## 四、用户权限

| 角色 | 能看到什么 | 能做什么 |
|------|-----------|----------|
| **Admin**（David） | 所有数据 | 系统设置、API Key、邮箱管理、创建/删除用户、查看全员数据 |
| **Manager** | 自己团队所有 SDR 的数据 | 分配 lead 给 SDR、查看团队表现 |
| **SDR** | 只看到分配给自己的客户和 lead | 录入活动、发邮件、管理自己的 pipeline |

SDR 之间数据互相隔离。Manager 可以重新分配 lead。


## 八、部署方案：全云端托管

**原则：不碰任何硬件。** 不用 Windows PC、不用 Mac mini、不用自己管服务器。所有东西都跑在云上，24 小时在线，自动备份，自动升级。

### 部署架构

| 组件 | 跑在哪 | 服务商 | 它做什么 |
|------|--------|--------|----------|
| 前端页面 | 云端 | **Vercel** | 用户打开浏览器看到的界面 |
| 后端 API | 云端 | **Railway** | 处理业务逻辑、对接 Apollo、调 AI |
| 数据库 + 向量搜索 | 云端 | **Supabase** | 存所有联系人、活动记录、向量数据（pgvector 内建） |
| 缓存 + 任务队列 | 云端 | **Upstash Redis** | Apollo 限流控制、后台任务 |
| 文件存储 | 云端 | **Supabase Storage** | 上传的附件、文档 |

### 为什么不用本地部署

| 方案 | 问题 |
|------|------|
| Windows PC | 电脑关机 = 系统停机；家庭网络不稳定；Windows 装 PostgreSQL + pgvector 折腾 |
| Mac mini | 要花 $599+ 买硬件；断电断网 = 停机；还是要自己维护 |
| Hostinger VPS | 需要 Linux 知识装数据库、配备份、打安全补丁——对技术小白是大坑 |

### 为什么选 Supabase

- 注册就能用，三步开好数据库，网页操作不碰终端
- pgvector（向量搜索）内建支持，不用额外安装
- 自动备份、自动升级、全球可访问
- 免费层 500MB 够 MVP 阶段用，之后 Pro plan $25/月

### SDR 怎么访问系统

系统部署完就是一个网址（例如 `https://crm.amazonsolutions.us`），SDR 用浏览器打开、登录就能用。手机、平板、任何电脑都行，不需要安装任何东西。
