## ⚡ 步骤 4：启用 Prompt Caching（部署前最后优化）

**目标**: 再砍 10-20% 成本，让月度稳定在 $20-30

### 什么是 Prompt Caching？

Claude 的 Prompt Caching 功能 — 把**每次调用都不变的 prompt 内容**（比如系统提示词、公司介绍、输出格式要求）缓存到 Anthropic 服务器。

- 缓存的 token 费用：**原价的 10%**
- 缓存有效期：5 分钟（连续调用会续期）
- 非常适合 SDR 批量处理客户场景

### 工作量
⏱️ 20-30 分钟

### 给 Claude Code 的指令

```
我要启用 Anthropic Prompt Caching 来降低 AI 调用成本。
Claude API 支持 cache_control 参数，能让固定的 prompt 
只收 10% 的费用。

要做的：

1. 检查所有调用 Claude API 的地方，识别哪些部分是"每次都一样"的：
   - AI 研究报告：系统提示词 + 公司介绍模板 + 输出格式
   - AI 邮件起草：系统提示词 + 公司风格指南
   - 语义搜索：系统提示词

2. 把这些固定部分加上 cache_control 标记，
   使用 Anthropic SDK 的 ephemeral 缓存类型：
   
   messages = [
       {
           "role": "system", 
           "content": [
               {
                   "type": "text", 
                   "text": "固定的系统提示词...",
                   "cache_control": {"type": "ephemeral"}
               }
           ]
       },
       {"role": "user", "content": "动态内容..."}
   ]

3. 把系统提示词抽出来作为 Python 常量或配置文件
   （便于维护和审查）

4. 在 ai_usage_log 表加字段：
   - cache_read_tokens (从缓存读的 token 数)
   - cache_write_tokens (写入缓存的 token 数)
   
   计费公式调整：
   - cache_write: 原价 1.25 倍
   - cache_read: 原价 0.1 倍

5. Admin 面板显示缓存命中率：
   "本月缓存节省: $X.XX (XX% hit rate)"

6. 完成后跑一遍测试，确认 API 响应正常

请告诉我哪些 prompt 适合缓存，让我审查确认后再执行。
```

### 验证
- 批量生成 10 个客户的研究报告
- 查看日志应该看到：
  - 第 1 次：cache_write 有值
  - 第 2-10 次：cache_read 有值，cache_write = 0
- 成本应该明显下降
